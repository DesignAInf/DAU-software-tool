# Prototypes

Research prototypes from *Design for Entropy* are grouped here with consistent folder names. The book-facing `README.md` at the repository root is unchanged.

| Folder | Description |
|--------|-------------|
| `battery_design_fep` | Inverse-design battery case study (Python + SciPy). |
| `beck_ramstead_inversion` | Beck–Ramstead inversion demo (Vite / React). |
| `beck_ramstead_inversion_2` | Variant inversion demo (Vite / React). |
| `beck_ramstead_inversion_3` | Variant inversion demo (Vite / React). |
| `dau_model_rigid_designer` | pymdp-based rigid-designer scenario. |
| `dau_model_rigid_user` | pymdp-based rigid-user scenario. |
| `dau_v2` | DAU v2 numpy simulation (CLI). |
| `dau_v2_dashboard` | DAU v2 Flask dashboard + browser UI. |
| `dau_v3_dashboard` | DAU v3 Flask dashboard + multi-profile analysis. |
| `designer_artifact_user` | Packaged Designer–Artifact–User model. |
| `emergent_conduit_fep` | Emergent conduit FEP package. |
| `joule_active_inference_model` | Joule / DMBD active inference experiments. |

Each subfolder that runs Python or Node includes a `requirements.txt` and/or `package.json` and a `README.md` (or legacy doc name) with run instructions.
